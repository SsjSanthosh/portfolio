About

Overwatch Heroes is an API focused project that I built to be able to get all the information about these characters in one place. I realized there was no current API with the latest stats, so I went ahead and built one! Of course, I also built a react client to showcase the data in a meaningful way, I am first and foremost a frontend dev. Using Django and DRF to build the backend API was an experience for sure, I hit quite a bit of stumbling blocks and had to do a lot of googling but in the end, it came together quite nicely. I am hoping to make this an overwatch one-stop-shop for players, especially the API, with more info about maps, plot, comics and more!

Questions

Why build this?
I LOVE overwatch. I have always wanted to do something with it, but I never knew what. I realized there was no current for all the heroes, so I got to work! I wanted to flex my django muscles and come up with a simple but effective API so anyone can use this to get info.
Why django for the backend?
I worked with Django in my previous internship and had some famialirity with it, and it's easy to get going with it's ORM and DRF makes it a breeze to build a REST API after you've gotten over the initial boilerplate and framework guidelines.
Was this your idea?
Yes! Everything from design to deployment, I did it from scratch and loved every minute of it.
Is this app complete?
Nope. I have a lot of ideas where I want to take this. Including a player search, adding videos, and more!
Are the assets hosted online or on the server?
All assets are hosted on the server. So many times, the online resources move and the API is rendered useless, so I pulled every image,fact and stat from the wonderful wikis and put them up on the server.
